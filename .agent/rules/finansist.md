# Правила проекта II Финансист (ii_finansist)

Все действия в рамках этого проекта должны следовать этим стандартам:

## 1. Архитектура и Код
- **Clean Architecture**: Разделение на `data`, `domain` и `presentation` слои.
- **Flutter Best Practices**: Использовать `flutter_lints`, следовать правилам именования Dart.
- **Документация**: Каждая новая функция или класс должны иметь комментарии о предназначении.

## 2. Безопасность и Git
- **Секреты**: Никогда не коммитить `.pem`, `.keystore`, `.env` или файлы с паролями.
- **GitHub**: После каждой значимой фичи или в конце сессии обязательно запускать воркфлоу синхронизации `sync-github`.

## 3. Коммуникация
- **Язык**: Вся коммуникация и документация (кроме кода) ведется на **русском языке**.
- **Стандарты**: Работа должна выполняться грамотно, четко и скрупулезно.

## 4. Специфика проекта
- Проект `ii_finansist` является основным приложением.
- Проект `finansist_codex` (корневой каталог) является хранилищем документации и вспомогательных скриптов.
